AI做法律咨询的同时，顺手把客户管理和知识管理做了

本文目录 知识管理最致命的设定：它发生在事后 分类发生在行动的时候，不发生在事后的冲动里 三个层面的自动沉淀 第一个层面：每一个咨询自动留痕 第二个层面：客户习惯自己积累 第三个层面：年底拿来挖趋势 不用建复杂的分类体系 几个容易踩的坑 这个东西有边界

有个问题我想问所有同行：你上一次认真做知识管理，是什么时候？

大部分人的答案——断断续续搞过，后来就停了。我也不例外。过去十年里，我至少启动过四五次"这一次一定要系统整理"的知识管理计划，最长的坚持了大概两个月。

不是懒。是真没力气。做完一天的咨询、审完合同、开完庭，你让我再打开一个笔记本"总结提炼"——杀了我吧。

但最近半年有件奇怪的事：我没有刻意做过一次知识管理。但我的知识管理做得比过去十年都好。

不是变勤快了。是这个动作被"顺手"掉了。准确地说，被 Agent 顺手掉了。

知识管理最致命的设定：它发生在事后

你仔细想——传统知识管理有个根子上的问题：干活和整理是两套动作。

你花一小时解答了一个股权回购的咨询，这件事在脑子里已经结案了。然后你告诉自己"我要把它归档整理"。这意味着你得重新打开案子、回忆思路、写成总结、归到某个分类下——这一套动作需要的意志力，远超"关掉窗口去刷手机"。

所以大部分人的知识管理不是死在方法上，是死在"等我有空了再整理"这个永远不会来的空档上。

但这个问题的解法，可能不是你想象的"更自律一点"。解法是把干活和整理变成一件事。

分类发生在行动的时候，不发生在事后的冲动里

用 Agent 之后，我所有的工作——写文章、审合同、做咨询、分析案件——都遵循同一套目录结构：

主题文件夹/
├── input/    ← 你扔进来的原始材料
├── scratch/ ← 中间过程，大纲、草稿、笔记
└── output/  ← 最终交给客户的东西

这套结构不是事后建的。是 Agent 在收到任务的那一刻自己建好的。

上周有个客户问我一个建设工程的结算争议。我把邮件和合同扔进 input/，Agent 在 scratch/ 里跑研究，扫了知识库里两本建工司法解释的理解与适用，翻了最近三年的类案，最后在 output/ 里出了一个结构化分析意见。

整个过程我一次都没有额外"整理"过。半年后我要找"那个结算争议当时怎么分析的"——东西就在那里，整整齐齐。

分类发生在你干活的那一刻，不发生在你写年终总结的那个下午。

三个层面的自动沉淀

如果说"咨询自动归档"只是帮你省了整理时间，后面两个层面才是真正值钱的东西。

第一个层面：每一个咨询自动留痕

你先回忆一下：去年你解答过多少法律咨询？其中股权类的有几个？建工类的有几个？劳动争议有几个？

能答出来的律师，我猜不到 10%。不是记性差。是以前从来没有人要求你在解答的同时做记录。你要么答完就完了，要么随手记在某个地方——下次要找根本想不起来记在哪了。

Agent 不是这样工作的。每一次咨询跑完，原始材料、研究路径、分析意见——都在那个文件夹里。需要的时候回头看，不需要的时候它安静待着。

今年年初我问了 Agent 一句："我去年解答的咨询里，哪些类型在增长？"

它把过去半年的咨询目录扫了一遍，告诉我：建工结算争议和股权回购在明显上升，劳动争议的量基本持平。这个信息本身，就值回了我搭这套东西的时间。

第二个层面：客户习惯自己积累

每个客户都有偏好。你朦朦胧胧是知道的。

你知道 A 公司法务总监是诉讼出身，每次看意见都喜欢附带相关案例。B 公司老板做投资的，习惯看表格对比，抗拒长篇大论。C 客户曾经因为你写了一句"风险可控"发飙——他说"可控是什么意思？你敢打包票？"此后你每次给 C 客户写意见，措辞都格外小心。

这些"知道"，以前存在你脑子里。模糊，不系统，偶尔会忘。

用 Agent 的办法是：给每个重要客户建一个几十行的"习惯文档"。

我第一次审 A 公司的合同时，跟 Agent 说了一句："这家法务喜欢附带案例，以后审这家合同，意见后面自动附两三个相关类案。"它就记下来了。下次再审 A 公司的合同，它自己会先去读这个文档。

半年后 A 公司那个文档里多了十几条类似的东西。它不是"我写给自己的备忘录"，是 Agent 每次干活之前会去读的东西——比存进脑子里靠谱多了。你半夜被紧急叫醒时，脑子里的东西不一定能调出来，但 Agent 一定会读。

第三个层面：年底拿来挖趋势

前面说的那个"去年咨询类型变化"的统计，就是第三层的例子。

你做律师这些年，产出了多少东西？解答了多少咨询？审了多少合同？出了多少法律意见？这些都是数据。只是以前没人帮你把它们串起来。

今年底你可以问 Agent："把我今年解答的所有法律咨询做分类，按类型统计数量，跟去年对比。""审过的合同里，最常见的修改点是什么？""今年出现的、去年没有的新类型问题有哪些？"

这些问题问完，你就不是在"年终总结"了，你是在挖矿。而且这个矿是你自己一天一天的工作堆积出来的，不用额外采集。

你一直在生产数据。缺的只是一套让数据自动落盘的东西。

不用建复杂的分类体系

你可能会想：那我是不是要先花一个月设计一个完美的知识分类体系？

千万别。这是另一个知识管理失败的主要原因——还没开始干活，先被自己的分类体系困住了。

我的做法极其简单：文件夹名就是分类。{日期}_{主题}，没了。

因为 AI 不需要你把东西按杜威十进制排好。它只需要一个可以遍历的结构。你设计分类的目的是给 AI 看，不是给你看。

以前你整理知识是为了自己将来能检索到。现在你整理知识是为了让 AI 能检索到。这两个目的完全不同。AI 的算力跟你不一样，按自己的习惯去设计，反而在限制它。

几个容易踩的坑

别一开始就追求完美。分类是长出来的，不是设计出来的。你现在的文件夹结构，大概率跟你三年前的业务范围完全不同。不重要，让它慢慢长。

知识管理必须嵌入工作流，不能叠加在工作流之上。如果你在做一个任务的同时，还需要另一个系统去记录心得——这个设计一定哪里不对。知识沉淀是正事的副产品，不是额外的作业。

不用你自己记住。这是心态上最大的调整。以前你不敢依赖系统，是因为任何系统都需要你去维护——维护不动了，它就死了。但 Agent 这套东西是自维护的：你每一次用它干活，它就在自动丰富自己。你甚至不需要知道它记住了什么，只要相信它记住了就行。

这个东西有边界

不是所有工作都适合这样自动沉淀。生效的前提是：高频重复、流程相对固定。

法律咨询、合同审核、写文章、案件分析——天然适合。代理词、法律意见书这种每件都不同的，也能沉淀其中的规律性部分。但纯粹的一次性研究，不值得搭自动归档。

投入也不大。大概花半个小时，想清楚你主要的几类工作场景，给每个场景定一个文件夹模板。Agent 每次启动自动建。后面零维护。

这整件事的底层逻辑是一句话：让你的工作方式变成"每次用都在做加法"，而不是"每次用完就归零"。

四明山法师 AI 夜校第三期，"工作区设计"就是一整节课的内容。从前两期的反馈来看，这是学员觉得最值回票价的一个模块——搭完之后，它就开始自己工作了，不需要你再额外投入。

前两期二十多个法律人学员，文科生、代码零基础、连终端都没打开过的都有。两节课后都在自己搭工作区。百分之百的作业完成率，不是我逼的，是搭出来东西真的能用、真的有爽感。

第三期正在招生。如果"每次用完就归零"这个状态你很熟悉，可以过来看看。
