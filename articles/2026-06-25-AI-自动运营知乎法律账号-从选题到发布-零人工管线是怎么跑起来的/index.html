AI 自动运营知乎法律账号：从选题到发布，零人工管线是怎么跑起来的

去年底搭了一个东西：一个 Agent，每天自动去知乎上找法律问题，拉回本地用我的知识库研究，写出回答，推到知乎上。

从设好规则那天起，我手动参与的次数是零。

这篇文章就是把它拆开给你看。不是为了让你照搬，是想让你看见：当 AI 不是"工具"而是"管线"的时候，它能做的事比你想象的多得多。

这条管线长什么样

四步：

发现选题 → 本地研究 → 撰写回答 → 发布到平台

每一步都是 Agent 自动跑的，人只在一开始设过规则。

第一步：发现选题

Agent 每天定时打开知乎，按我预设的标签——建设工程、房地产、公司法、合同纠纷——扫描当天新增的法律类问题。过滤掉已经有大量专业回答的（你去硬碰没意义），找那些"有人问、有一定关注、但还没有律师认真回答"的问题。

筛选规则一开始就告诉它了：只看法律实务类问题，不要围观型热点（"怎么看待某某事件"），不要泛泛的求助帖（"我该怎么办"但没有任何具体事实）。优先选有具体事实描述的问题——说明提问者是真实遇到了法律困境。

每天大概筛出两三个候选，按"回答价值"排序，选最高的两个进下一步。

这里的"回答价值"有一套打分逻辑：问题的搜索流量潜力（标题里有没有精准的法律关键词）、提问的事实密度（够不够支撑专业分析）、竞争程度（已有回答的质量和数量）。

第二步：拉到本地做研究

选定问题后，Agent 把问题文本拉回本地。

这是最关键的一步。也是整条管线和"你打开 ChatGPT 输入问题"的本质区别。

Agent 不是直接开始写答案。是按我预设的法律研究路径跑一遍：

  1. 识别法律领域（建工/公司/劳动/知产……）
  2. 去本地知识库检索这个领域的重点资料——法律、司法解释、理解与适用、实务书籍
  3. 如果涉及具体法条，通过 MCP 去商用数据库核验是否现行有效
  4. 检索最近三年的类案，确认裁判口径
  5. 全部跑完，才开始组织回答

整个过程大概三到五分钟。如果是我自己查，光翻那几本理解与适用就不止这个时间——所以我过去大概率不会全查。但 Agent 会。

第三步：按我的方式写回答

研究做完，开始写。

这里有个很容易被忽略的东西：回答不是"AI 风格"的，是按我的写作偏好来的。

因为在管线里预设了回答的写作规则——来源于我之前手动写的几十篇知乎回答提炼出来的特征：

  • 不要"结论先行"四个字，但结构上结论先行
  • 先给判断再给依据，不要铺垫
  • 法条带全名和条文编号，不能只写"根据相关法律规定"（知乎法律回答最容易被喷的点）
  • 对不确定的结论，直接标注"这一点实践中存在争议，有两种观点"——知乎读者对有边界的回答反而更信任
  • 自然地引导关注，不生硬

写完之后 Agent 自己跑一遍 humanizer-zh，去掉明显的 AI 痕迹。

第四步：发布

最后一步简单：Agent 打开知乎，粘贴回答，提交。

流程跑完我会收到一条通知："已发布：{问题标题} -> {链接}"。一周集中扫一眼，看看有没有需要改的。绝大多数时候不需要。

几个技术细节

关于定时触发。不是靠人每天去点"开始"。一个 cron 任务设了每天早上九点和下午两点各跑一次。上午扫新问题，下午有合适的就再发一篇。

关于失败处理。管线不是每次都顺利。知乎的反爬有时候会拦截，知识库检索偶尔超时，MCP 数据库请求可能失败。给 Agent 设的规则是：遇到失败记录到日志，自动重试一次，再失败就跳过，发通知给我。不卡住整个管线。

关于质量。说实话不是每一篇都是精品。大概 70% 的回答"能用"——准确、有依据、比知乎上大部分随手回答强。偶尔有特别好的，因为那个问题恰好落在我知识库覆盖最深的领域，Agent 调用的资料特别对口。也有个别需要手动改的，平均一周不超过一篇。

常见坑

坑一：MCP 数据库不是每次都通。商用法律数据库的 API 偶尔抽风。如果依赖它做法规时效性核验，要做好降级——备一条"连不上就标注'时效性未核验，仅供参考'"的规则。

坑二：平台反爬。知乎对自动化操作比较敏感。不要太频繁，一天两篇以内，间隔至少四小时。即使这样，大概 5% 的发布会失败，Agent 会自动记录、延后重试。

坑三：知识库的偏见。你的知识库决定了回答的质量上限。如果知识库里只有建工领域的资料，Agent 碰上公司法问题就只能靠模型通用知识来答——那个质量跟你手动查资料差不多。知识库是需要养的，不是一次性塞进去就完了。

坑四：法律风险。这条管线生成的所有内容，在法律上仍然是你的表达。你对内容的真实性、准确性、合规性负责，不能因为"是 AI 写的"就免责。我的做法：回答末尾标注"以上分析仅供参考，具体案件请咨询律师"，每周抽时间抽查已发布内容。

效果

这条管线跑了快一年，几个数字：

  • 发布频率：大概一天两篇
  • 人工介入：大概每两周手动改一篇，其他时间完全不碰
  • 流量贡献：知乎进来的关注和咨询，占线上获客的大概四分之一

没有这条管线之前，我一周大概写两篇知乎回答，一年一百篇左右。有了它，一年四五百篇。不是"我变快了"，是我根本不在这个循环里了。

这件事的真正含义

很多人问我搭这条管线花了多久。三四个晚上。

但这不重要。重要的是：这条管线不是我写的代码，是我做的决策。

我决定：知乎运营值得做 → 我设了选题标准和写作规则 → 我定了质量底线和风险控制 → 剩下的交给 Agent 跑。

从"我自己来做这些事"到"我把这些事定义好让它来做"——这就是这篇文章真正想讲的。

以前你看到日更的法律公众号，会想"这个团队得有好几个人"。以后你看到高质量高频更新的内容账号，大概可以假设——背后是一个人，配了一套管线。

想做小红书也是一样的

这条管线目前只跑在知乎上。不是不能做小红书，是我的读者群暂时不在那儿。如果你做 C 端业务——婚姻家事、劳动仲裁、消费者维权——小红书可能是更好的选择。管线逻辑完全一样：选题发现 → 研究 → 撰写 → 发布。唯一多一步：小红书需要配图，Agent 自动生成封面就行。

怎么搭、每一环怎么设计、质量把控的规则怎么设——四明山法师 AI 夜校第三期有一整节课叫"内容管线设计"。

前两天一个往期学员跟我说，他搭了一条劳动法问答的管线跑在小红书上，三周涨了八百粉。

我说挺好的，但别涨太快——平台会以为你是营销号。
