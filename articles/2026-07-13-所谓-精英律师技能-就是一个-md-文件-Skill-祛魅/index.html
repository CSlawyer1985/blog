所谓"精英律师技能"，就是一个 .md 文件——Skill 祛魅

本文目录 Skill 的本质，是一份结构化提示词 打开一个看看 "精英"与否，看内容不看名字 改，才是真正的分水岭 Skill 不是越多越好

打开任何一个 Agent 产品的技能市场，你会看到一堆名字：精英律师、合同审查专家、法律写作大师、尽职调查专家……

名字很唬人。有些还要付费，有些要你配 API key 才能用。

你可能会想：这些技能到底是怎么做出来的？凭什么叫"精英"？里面是不是藏着什么我没见过的黑科技？

答案可能让你意外。

它们大多数，就是一个 .md 文件。

Skill 的本质，是一份结构化提示词

任何一个 Skill，不管名字多唬人，你点开它所在的文件夹看一眼。

里面都有一个 skill.md（有的产品叫 SKILL.md）。

这个文件就是一份说明书。它写清楚了这个技能是干嘛的、怎么干、输出什么格式、遵循什么规则、禁止做什么。

你在对话里加载一个 Skill，本质是什么？

是模型在干活之前，先去把这份 skill.md 读了一遍。

换句话说，"加载技能"和"你把这份 skill.md 里的文字手动复制粘贴到对话框里"，效果是一样的。

Skill 做的事情，只是把这段提示词预先存好了，按需加载，不用你每次手动粘贴。仅此而已。

打开一个看看

拿一个常见的法律服务 Skill 举例。打开它的 skill.md，你会看到这些内容：

  • 角色定位：资深商事律师，擅长合同、公司、建工
  • 推理方法：演绎推理、类比推理、体系解释
  • 输出格式：结论先行、分点展开、法条标注出处
  • 法律冲突规范：上位法优于下位法、新法优于旧法
  • 禁止行为：编造法条、模糊归因、空洞升华

这些内容，和你在系统提示词里自己写的东西，是同一类东西。

差别只在于：别人替你写好了，你直接用。

但"别人替你写好了"，不等于"写得好"。

"精英"与否，看内容不看名字

一个 Skill 叫"精英律师"，不代表它的 skill.md 写得精英。

名字是营销，内容才是实质。

判断一个 Skill 值不值得用，方法很简单：打开它的 skill.md，从头到尾读一遍。读的时候问自己四个问题：

推理步骤清不清楚？ 如果它只是堆砌"专业、严谨、全面"这类形容词，没有具体的推理路径，那它基本上是个空壳。

输出格式符不符合你的场景？ 每个 Skill 的输出格式是写它的人定的。你做的是中国非诉业务，它按美国诉讼的格式输出，用起来就拧巴。

有没有考虑中国法律的特点？ 很多 Skill 是国外开发者做的，法条引用、文书体例、合规口径都按普通法系来。直接用，水土不服。

禁止行为合不合理？ 一个好的 Skill 会明确写"不能做什么"。只写"要做什么"、不写"不能做什么"的，往往是没踩过坑的人写的。

读完这四个问题，你就知道：这个 Skill 是真材实料，还是个花架子。值不值得用，值不值得改。

改，才是真正的分水岭

大多数人用 Skill，停留在"加载别人写好的"。

真正的门槛，是"改"。

举个例子。你加载了一个法律服务 Skill，用了一次发现：它输出的 Word 文档，标题是左对齐的，你觉得不好看，想要居中加粗红色。

你有两个选择。

选择一：每次用完，手动在 Word 里把标题改成居中加粗红色。

选择二：跟它说一句——"以后你排版，标题一律居中加粗红色，这是强制要求，去改一下你的规则文件，记住。"

它就去改了。改的就是那个 skill.md。

下次你再加载这个 Skill，标题自动居中加粗红色。一劳永逸。

同样一个 Skill，别人加载的是原版，你加载的是改版。你们用的是同一个名字的技能，但出来的东西不一样。

这才是相对竞争力的来源。

不是你用了别人没有的 Skill，是你把同一个 Skill 改成了更适合自己的版本。这件事，别人替不了你。

Skill 不是越多越好

最后说几个边界。

第一，Skill 不是万能的。复杂场景里，一个 Skill 不够用，需要几个 Skill 组合起来。指望一个"精英律师"Skill 解决所有问题，是不现实的。

第二，Skill 也不是越复杂越好。一个好的 Skill 应该聚焦做一件事。一个 Skill 里什么都想管，最后什么都管不好。

第三，最重要的不是你有多少 Skill，是你能不能读懂、改对。

一个能打开 skill.md 读完、判断好坏、动手修改的人，比手握一百个 Skill 但从没打开过任何一个的人，强得多。

祛魅这件事的意义就在这里。

Skill 不是黑箱，不是魔法，更不是什么专家系统。它就是一段文字，一份说明书，一个 .md 文件。

你看得懂说明书，你就用得好 Skill。你会改说明书，你就能让 Skill 替你干更多的活。

就这一件事。没有更深的秘密。
